# Contributing

## Branching

- Create a feature branch from `main`.
- Use clear branch names, for example: `docs/update-faq-payment`.

## Pull Request Process

1. Keep each PR focused on one topic.
2. Follow `.github/PULL_REQUEST_TEMPLATE.md`.
3. Wait for at least one approval from CODEOWNERS.
4. Merge after CI and preview checks pass.

## Content Rules

- Follow `docs/style-guide.md`.
- Keep language neutral and practical.
- Prefer short paragraphs and numbered steps.
