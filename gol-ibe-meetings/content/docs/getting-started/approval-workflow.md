# How the approval workflow works

Meeting requests follow a configurable approval workflow before travel can be booked.

## Default flow

1. **Organiser** creates and submits a meeting request.
2. The system routes it to the **Approver** assigned to the organiser's cost centre.
3. The approver receives an email with a direct link to review the request.
4. The approver clicks **Approve** or **Reject** (with an optional comment).
5. The organiser is notified of the decision.
6. If approved, the travel agent can start booking flights and hotels.

## Multi-level approvals

For high-value meetings, you can configure a second-level approver who is notified after the first approver approves.

See [Set up approvers](/portal/configuration/setup-approvers).

## Timeouts and escalations

If an approver does not act within the configured deadline, the system can:
- Send a reminder email.
- Escalate to a backup approver.

See [Escalation rules](/portal/configuration/escalation-rules).
