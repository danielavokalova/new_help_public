# Meeting statuses explained

Every meeting request moves through a defined set of statuses.

## Status reference

| Status | Meaning |
|---|---|
| **Draft** | Created but not yet submitted. Only visible to the organiser. |
| **Pending approval** | Submitted and waiting for the approver to act. |
| **Approved** | The approver has confirmed the meeting. Travel can now be booked. |
| **Rejected** | The approver declined the request. The organiser can revise and re-submit. |
| **Booked** | Travel has been booked for at least one attendee. |
| **Completed** | The meeting end date has passed and all bookings are settled. |
| **Cancelled** | The meeting was cancelled by the organiser or an admin. |

## Transitions

```
Draft → Pending approval → Approved → Booked → Completed
                       ↘ Rejected → (organiser edits) → Draft
Any status → Cancelled
```
